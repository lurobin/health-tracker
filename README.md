## 核心逻辑流程

### 多因子推理链（优先级从高到低）

```
低血糖(心慌手抖) → 快碳需求 → 神经空虚 → 体重波动 → 膝盖状态
```

每个优先级会**短路**后续判断——高优先级信号未处理前，不进入下一层。

### 强度第一准则

```
压痛测试(阳性) ────→ 当日休息(无条件)
       │
       ↓
晨起RHR ≤72 ──────→ 正常训练
晨起RHR 73–75 ────→ 降强度(仅康复+步行)
晨起RHR ≥76 ──────→ 完全休息
```

**压痛测试优先级高于一切心率指标**——拇指按压胫骨粗隆 + 关节内侧间隙，任一处阳性自动触发休息日。

### 每日运行闭环

```
用户录入 → AI 读取 → 数据解读 → 问题定位(≤2个) → 调整建议(≤3条) → 生成次日文件
```

通过两个触发词控制：

| 触发词 | 行为 |
|:---|:---|
| `look` / 查看 | 读取当日数据 → 即时反馈（写入 feedback 文件） |
| `lock` / 总结 | 反馈 + Gap 分析 → 自动生成次日日记+训练文件 → 锁定当日 |

## 核心功能

### 📋 自适应训练系统

**三天轮换**：上肢推(A) / 上肢拉(B) / 全身循环(C)，配合核心四选一每日轮换

**模块状态管理**：每个模块有 4 种自适应状态

```
活跃(每天) → 降频(隔天) → 维持(周1-2次) → 休眠(暂停)
```

状态根据身体反馈自动流转，无需手动调整。

**最小有效剂量规则**：连续 2 天 RHR 升高或主观疲劳 → 执行最低合格线（核心×1组 + 上肢×1组 + 扳机），不追完整训练。

### 🩺 医学知识驱动

| 模块 | 参考框架 | 来源 |
|:---|:---|:---|
| 膝康复 | Jill Cook 腱病分期（Stage 0→3） | 《运动损伤临床指南》 |
| 扁平足康复 | 阶段零（TA松解）→ 阶段三 | 《练脚掌是最好的复健！》 |
| 动作膝负荷分级 | Traffic Light 系统（🟢🟡🔴） | NASM-CES 纠正性训练 |
| 步态纠正 | 髋主导步态 + Mayo Clinic 清单 | Gait & Posture 2019 |
| 足弓评估 | 舟骨沉降测试 + 足弓指数 | Brody 1982 / Murley 2009 |

### 🍽️ 饮食体系

训练日 / 休息日区分，可执行分量指标。主食一次做 300g 生重，分 6 份，每份 50g——量化到不需要秤。

### 📊 验证闭环（🟢🟡🔴）

| 信号 | 标准 | 处理 |
|:---|:---|:---|
| 🟢 | 周降 0.3–0.6kg | 继续 |
| 🟡 | 周降 >0.8kg + 疲劳 | 补碳+10g |
| 🔴 | 日降 >1kg + RHR升高 | 停练+补碳 |

### 🔄 疼痛响应系统（Traffic Light）

全部 20+ 动作按膝负荷分级，疼痛日自动过滤：

| 等级 | 示例 | 疼痛日处理 |
|:---|:---|:---|
| 🟢 安全 | 靠墙静蹲、TKE、核心 | 正常执行 |
| 🟡 谨慎 | 臀桥、腿弯举、悬挂 | 退阶或回避 |
| 🔴 替换 | 单车、步行>15min | 强制替换为 🟢 动作 |

## 设计理念

**1. 用户手动记录 > 设备自动数据**——手动记录本身就是建立知觉的过程。

**2. 模板驱动，非 prompt 驱动**——系统核心是 `Guides/` 下的结构化模板，AI 只是执行者。换一个 AI 模型，系统照常运行。

**3. 知识工程化**——18 本参考文献不是"墙上的装饰"，每条规则都对应一条可追溯的文献来源。

**4. 隐私优先**——个人数据完全存储在本地 `Private/`。不上传、不共享、不 SaaS。AI 读取本地文件，处理结果也写入本地。
